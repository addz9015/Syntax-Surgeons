Product Overview `r`n`r`n`r`n`r`n 1.1 Executive Summary `r`n`r`n`r`n VitalEdge+ is a clinician-facing intelligence dashboard that transforms the VitalEdge healthcare data warehouse into a proactive patient safety system. Built on an ETL pipeline, SCD Type 2 historical tracking, and a star schema data model, the platform adds six intelligence layers: real-time allergy-prescription conflict detection, AI/ML risk prediction with explainability, an Allergy Burden Score, a Doctor Prescribing Risk Profile, patient allergy cluster analysis, and an interactive What-If Simulator. The entire system is presented through a unified React + Power BI dashboard requiring no mobile or backend infrastructure beyond what the original VitalEdge project established. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 1.2 Problem Statement `r`n`r`n`r`n Adverse drug reactions caused by undetected allergy-medication conflicts represent one of the most preventable sources of patient harm in Indian hospitals. Existing systems store prescriptions and allergy records in separate silos with no automated cross-referencing. Clinicians rely on manual recall and fragmented paper records, while hospital administrators have no visibility into systemic prescribing risk patterns. Patients have no access to their own allergy history. `r`n`r`n`r`n `r`n`r`n`r`n VitalEdge+ addresses three compounding gaps simultaneously: the individual patient safety gap (missed conflicts at the point of prescribing), the institutional visibility gap (no system-wide risk analytics), and the explainability gap (ML predictions that clinicians cannot interpret or trust). `r`n`r`n`r`n `r`n`r`n`r`n`r`n 1.3 Vision `r`n`r`n`r`n To be the most trusted allergy-safety intelligence layer in Indian clinical data infrastructure — one that prevents medication harm before it occurs, surfaces systemic risk patterns to hospital leadership, and earns clinician trust through transparent, explainable AI. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 1.4 Scope — Dashboard Only `r`n`r`n`r`n VitalEdge+ is scoped as a unified intelligence dashboard. There is no mobile application, no real-time WebSocket infrastructure, and no patient-facing portal in this version. The event is written to the prescriptions table and triggers the conflict engine. `r`n`r`n`r`n`r`n Step 2 — Conflict Check: The Conflict Detection Engine queries Dim_Allergy (Is_Current = True) and Dim_Medicine (Allergen_Class, Cross_Reactive_With) and classifies the event as CRITICAL, WARNING, ADVISORY, or NONE. `r`n`r`n`r`n`r`n Step 3 — ML Scoring: The XGBoost model scores the event for allergy reaction probability. Feature Specifications `r`n`r`n`r`n`r`n 4.1 Allergy-Prescription Conflict Detection Engine `r`n`r`n`r`n Priority: P0 — Core Safety Feature `r`n`r`n`r`n The Conflict Engine is the foundational safety layer. It automatically cross-references every new prescription against the patient's current allergy profile in real time, classifying conflicts into three actionable severity tiers and blocking CRITICAL prescriptions before they can be dispensed. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Conflict Severity Classification `r`n `r`n Severity `r`n `r`n Trigger Condition `r`n `r`n Dashboard Action `r`n `r`n CRITICAL `r`n `r`n Prescribed drug's Allergen_Class directly matches patient's active allergen `r`n `r`n Red banner; prescription blocked; override requires justification `r`n `r`n WARNING `r`n `r`n Cross-reactive substance detected via Cross_Reactive_With field `r`n `r`n Orange alert; prescription flagged; clinician acknowledgement required `r`n `r`n ADVISORY `r`n `r`n Indirect family-class match or historical mild reaction on record `r`n `r`n Yellow notice; prescription proceeds; event logged for audit `r`n `r`n NONE `r`n `r`n No conflict found in current allergy profile `r`n `r`n Green confirmation; prescription approved silently `r`n `r`n`r`n `r`n`r`n`r`n`r`n Functional Requirements `r`n`r`n`r`n`r`n FR-1.1: For every prescription event, the engine SHALL query Dim_Allergy filtered to Is_Current = True for that Patient_ID. `r`n`r`n`r`n`r`n FR-1.2: The engine SHALL check Allergen_Class and Cross_Reactive_With from Dim_Medicine against all active allergen classes for the patient. `r`n`r`n`r`n`r`n FR-1.3: CRITICAL conflicts SHALL prevent prescription completion on the dashboard until a clinician override with documented justification is recorded. `r`n`r`n`r`n`r`n FR-1.4: All conflict outcomes SHALL be written to Fact_AllergyEvents with timestamp, conflict type, and the Allergy_Key version active at that moment (SCD Type 2 point-in-time reference). `r`n`r`n`r`n`r`n FR-1.5: The engine SHALL complete its check and return a result to the dashboard within 2 seconds. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.2 AI/ML Allergy Risk Prediction with Explainable AI `r`n`r`n`r`n Priority: P0 — Key Technical Differentiator `r`n`r`n`r`n The ML module predicts the probability of an adverse allergy reaction to a prescribed medication, even when no direct conflict record exists. Every high-risk prediction is accompanied by a SHAP-based explanation panel — surfacing the exact features driving the score in plain language that a clinician can understand and act on. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Model Design `r`n`r`n`r`n`r`n Primary Model: XGBoost classifier trained on allergy_events.csv with Reaction_Occurred as the binary target label. `r`n`r`n`r`n`r`n Feature Set: Patient demographics (Age, BMI, Comorbidities), allergy profile (Allergen_Class, Severity, number of active allergies), lab markers (IgE Level, Eosinophil Count, Flag), and prescription details (Drug class, Dosage, Ward). `r`n`r`n`r`n`r`n Explainability: SHAP TreeExplainer computes feature contribution values for every prediction. Predictions with ML_Risk_Score ≥ 0.75 trigger the XAI Panel on the dashboard. `r`n`r`n`r`n`r`n Retraining: Batch retraining on newly confirmed reaction events using the allergy_events pipeline. `r`n`r`n`r`n `r`n`r`n`r`n`r`n XAI Panel — Dashboard Specification `r`n`r`n`r`n For every high-risk prescription, the dashboard displays: `r`n`r`n`r`n`r`n Risk Score gauge (0–100) with colour-coded zones: Green (< 50), Amber (50–74), Red (≥ 75). `r`n`r`n`r`n`r`n Top 3 SHAP Contributors listed as plain-language cards, e.g. 'IgE Level of 340 kU/L is 3.2x the normal upper limit and is the strongest driver of this risk score.' `r`n`r`n`r`n`r`n Recommendation panel: suggested alternative drug class with lower allergen risk, drawn from Dim_Medicine. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Functional Requirements `r`n`r`n`r`n`r`n FR-2.1: The model SHALL produce a risk score in [0, 1] for every prescription event, stored in Fact_AllergyEvents.ML_Risk_Score. `r`n`r`n`r`n`r`n FR-2.2: SHAP values SHALL be computed for all predictions with score ≥ 0.75 and rendered in the XAI Panel within the same dashboard response. `r`n`r`n`r`n`r`n FR-2.3: Model metrics (F1, AUC-ROC, Precision, Recall) SHALL be visible on the Data Quality & Model Monitor page of the dashboard. `r`n`r`n`r`n`r`n FR-2.4: The model SHALL NOT surface a score without an accompanying explanation for any high-risk prediction — unexplained ML outputs are a compliance violation in this system. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.3 Allergy Burden Score `r`n`r`n`r`n Priority: P0 — Novel Metric / Innovation Highlight `r`n`r`n`r`n The Allergy Burden Score (ABS) is a proprietary composite index (0–100) that quantifies a patient's cumulative allergy risk at a single glance. It is computed per patient on every ETL run and stored in Dim_Patient. No commercially available healthcare system currently produces this metric — it is a direct innovation contribution of VitalEdge+. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Score Formula `r`n`r`n`r`n ABS = ( W1 × Severity_Score ) + ( W2 × Allergen_Count ) + ( W3 × Reaction_History ) + ( W4 × Lab_Marker_Score ) `r`n`r`n`r`n `r`n `r`n Component `r`n `r`n Description `r`n `r`n Weight (W) `r`n `r`n Data Source `r`n `r`n Severity_Score `r`n `r`n Weighted average of active allergy severities: MILD=1, MODERATE=2, SEVERE=3, LIFE_THREATENING=4; normalised to 0–25 `r`n `r`n 0.30 `r`n `r`n allergies.csv `r`n `r`n Allergen_Count `r`n `r`n Number of distinct active allergen classes; capped at 5; normalised to 0–25 `r`n `r`n 0.25 `r`n `r`n allergies.csv `r`n `r`n Reaction_History `r`n `r`n Count of confirmed past reactions (Reaction_Occurred=True); log-scaled; normalised to 0–25 `r`n `r`n 0.25 `r`n `r`n allergy_events.csv `r`n `r`n Lab_Marker_Score `r`n `r`n Composite of IgE Level and Eosinophil Count z-scores relative to population mean; normalised to 0–25 `r`n `r`n 0.20 `r`n `r`n lab_results.csv `r`n `r`n`r`n `r`n`r`n`r`n`r`n Dashboard Usage `r`n`r`n`r`n`r`n Patient Profile Page: ABS displayed as a gauge widget with risk band label (LOW / MODERATE / HIGH / CRITICAL). `r`n`r`n`r`n`r`n Hospital Overview Page: Distribution of ABS across all patients, filterable by ward, age group, and comorbidity. `r`n`r`n`r`n`r`n Trend View: ABS plotted over time using SCD Type 2 version history — shows whether a patient's burden is increasing or resolving. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.4 Doctor Prescribing Risk Profile `r`n`r`n`r`n Priority: P1 — Institutional Impact Feature `r`n`r`n`r`n The Doctor Risk Profile flips the analytical lens from patient-centric to system-centric, revealing which clinicians and wards have the highest rates of conflict-positive prescriptions, overrides, and post-prescription reactions. This module directly addresses the Real-World Impact judging criterion by producing insights that a hospital administrator could act on immediately. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Profile Metrics per Doctor `r`n `r`n Metric `r`n `r`n Definition `r`n `r`n Source `r`n `r`n Conflict Rate (%) `r`n `r`n % of prescriptions that triggered a CRITICAL or WARNING conflict `r`n `r`n prescriptions.csv + allergy_events.csv `r`n `r`n Override Rate (%) `r`n `r`n % of conflict prescriptions where Override_Issued = True `r`n `r`n allergy_events.csv `r`n `r`n Reaction Rate (%) `r`n `r`n % of override prescriptions where Reaction_Occurred = True `r`n `r`n allergy_events.csv `r`n `r`n High-Risk Prescription Count `r`n `r`n Total prescriptions with ML_Risk_Score ≥ 0.75 `r`n `r`n allergy_events.csv `r`n `r`n Risk Tier `r`n `r`n Computed tier: GREEN / AMBER / RED based on composite of above metrics `r`n `r`n Derived `r`n `r`n`r`n `r`n`r`n`r`n`r`n Dashboard Views `r`n`r`n`r`n`r`n Leaderboard Table: All doctors ranked by Conflict Rate, with colour-coded Risk Tier badges. Sortable and filterable by ward and time period. `r`n`r`n`r`n`r`n Ward Heatmap: A heatmap of Conflict Rate × Override Rate across all wards — highlights systemic risk pockets, not just individual clinicians. `r`n`r`n`r`n`r`n Drill-Down: Click any doctor row to see their full prescription history with allergy event overlay. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.5 Allergy Cluster Analysis `r`n`r`n`r`n Priority: P1 — ML Depth Feature `r`n`r`n`r`n K-Means clustering segments patients into natural risk groups based on their allergy profile, lab markers, and Allergy Burden Score. This unsupervised ML component demonstrates machine learning depth beyond the supervised risk predictor and provides the hospital with clinically meaningful patient cohorts for targeted interventions. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Clustering Design `r`n`r`n`r`n`r`n Algorithm: K-Means (k=4, determined by elbow method on the synthetic dataset). `r`n`r`n`r`n`r`n Feature Space: Allergy Burden Score, number of active allergen classes, IgE Level (normalised), Eosinophil Count (normalised), Age, number of comorbidities. `r`n`r`n`r`n`r`n Cluster Labels (expected from synthetic data distribution): `r`n`r`n`r`n`r`n Cluster A — Low-Risk Single-Allergen: Young patients, 1 mild allergy, normal lab markers. `r`n`r`n`r`n`r`n Cluster B — Moderate Poly-Allergic: Middle-aged, 2–3 allergens, mildly elevated IgE. `r`n`r`n`r`n`r`n Cluster C — High-Risk Complex: Older patients, 3+ severe allergies, elevated IgE and eosinophils, multiple comorbidities. `r`n`r`n`r`n`r`n Cluster D — Lab-Elevated Undiagnosed: Normal allergy record but significantly elevated IgE — potential undiagnosed sensitisation, flagged for clinical review. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Dashboard View `r`n`r`n`r`n`r`n 2D PCA scatter plot of clusters rendered in the analytics panel with colour-coded cluster membership. `r`n`r`n`r`n`r`n Cluster summary cards: count of patients per cluster, average ABS, most common allergen class, most common comorbidity. `r`n`r`n`r`n`r`n Cluster D (Undiagnosed) patients are surfaced as a priority action list for the clinical team. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.6 What-If Simulator `r`n`r`n`r`n Priority: P1 — Demo Impact Feature `r`n`r`n`r`n The What-If Simulator is an interactive React panel that allows a clinician or judge to input any patient ID and medicine name and receive an immediate, complete safety assessment — conflict level, ML risk score, SHAP explanation, Allergy Burden Score, and recommended alternatives. It is the single most powerful demo tool in the system and is specifically designed to make every capability of VitalEdge+ visible in one live interaction. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Simulator Flow `r`n`r`n`r`n`r`n Step 1 — Input: User selects a Patient ID from the dropdown and types or selects a Medicine Name. `r`n`r`n`r`n`r`n Step 2 — Conflict Check: The Conflict Engine fires against the patient's current Dim_Allergy records and returns the severity classification within 1 second. `r`n`r`n`r`n`r`n Step 3 — ML Score: XGBoost model scores the hypothetical prescription. SHAP values are computed if score ≥ 0.75. `r`n`r`n`r`n`r`n Step 4 — Result Panel: Dashboard renders a unified result card showing: Conflict Level badge, ML Risk Score gauge, top SHAP contributors, Allergy Burden Score, and a recommended safe alternative drug class. `r`n`r`n`r`n`r`n Step 5 — Audit: The simulation is logged as a SIMULATION event type in Fact_AllergyEvents for full traceability. `r`n`r`n`r`n `r`n`r`n`r`n`r`n Functional Requirements `r`n`r`n`r`n`r`n FR-6.1: The simulator SHALL complete all checks and render the full result panel within 2 seconds of user input. `r`n`r`n`r`n`r`n FR-6.2: The simulator SHALL be clearly labelled as a simulation — no simulation event SHALL affect live patient records. `r`n`r`n`r`n`r`n FR-6.3: The recommended alternative drug class SHALL be drawn from Dim_Medicine filtered to Requires_Allergy_Check = False and a different Allergen_Class than the conflicting drug. `r`n`r`n`r`n `r`n`r`n`r`n`r`n 4.7 Data Quality Scorecard & Model Monitor `r`n`r`n`r`n Priority: P2 — Technical Credibility Feature `r`n`r`n`r`n A dedicated dashboard page that makes the engineering rigour of VitalEdge+ visible to technical judges. Extended Data Model `r`n`r`n`r`n`r`n 5.1 Schema Overview `r`n`r`n`r`n The VitalEdge+ schema extends the original star schema with two new tables (Dim_Allergy and Fact_AllergyEvents), three extended columns on Dim_Medicine, and a new computed column (Allergy_Burden_Score) on Dim_Patient.
